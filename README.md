## Branch 규칙
- 기능 구현 : feat/00 브랜치 사용
- 리팩토링 : refactor/00 브랜치 사용
- 버그 수정 : fix/00 브랜치 사용
- 개발 환경 세팅 : chore/00 브랜치 사용
</br>

## Commit 유의 사항
> 💡 **Tip**: 개인 작업 브랜치 -> develop 브랜치 -> main

작업 시작시 항상 main 브랜치의 내용을 develop으로 pull

pull request는 develop 브랜치를 거쳐 main으로 보내기!

개발 완료된 사항만 main 브랜치로 merge하기
